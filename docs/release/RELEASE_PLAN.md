# Release Plan

本文件定义发布计划模板。每次正式发布、灰度发布、全量发布或重要配置变更前，必须由 DevOps/SRE 智能体产出发布计划，主会话组织用户确认。

## 发布门禁

以下情况禁止发布：

- P0/P1 缺陷未关闭。
- 核心路径未通过验证。
- 回滚方案缺失或不可执行。
- 监控、错误日志、核心埋点缺失。
- 数据库迁移没有验证或没有兼容策略。
- 用户或指定验收人未确认发布范围。

## 灰度转全量条件

灰度转全量前必须满足：

- 灰度观察期内无 P0/P1。
- 错误率、延迟和核心业务指标无明显异常。
- 用户反馈无阻塞问题。
- 回滚通道仍然可用。
- 主会话已向用户汇报灰度结果并获得确认。
