# 国际化 (i18n) 规范

本文件定义多语言支持、文本管理、动态加载和本地化要求。

## 核心原则

- 所有用户可见文本必须支持多语言，不写死原文。
- 语言切换不应刷新页面。
- 日期、时间、数字、货币格式必须本地化。
- RTL（从右到左）语言必须正确适配。

## 适用场景

需要国际化的产品：

- 面向海外用户的 Web/App
- 面向多地区的 B2B/B2C 产品
- 未来可能出海的产品（预留 i18n 能力）

## 文本管理

### 文本提取规范

- 所有硬编码文本必须提取到语言包，不允许散落在代码中
- 文本键命名：`模块_页面_组件_用途`

示例：

```json
{
  "common_button_submit": "提交",
  "common_button_cancel": "取消",
  "common_button_delete": "删除",
  "common_toast_success": "操作成功",
  "common_toast_error": "操作失败",
  "user_login_title": "登录",
  "user_login_email_placeholder": "请输入邮箱",
  "user_login_password_placeholder": "请输入密码",
  "user_profile_title": "个人资料"
}
```

### 语言包结构

```
locales/
  zh-CN.json      # 简体中文（默认）
  en-US.json      # 美国英语
  zh-TW.json      # 繁体中文
  ja-JP.json      # 日语
  ko-KR.json      # 韩语
  ...
```

### 文本格式

- 值必须是字符串，不允许拼接字符串：`"订单号：" + orderId` → `"order_detail_id": "订单号 {orderId}"`
- 支持插值变量：`"欢迎 {name}"`、`"剩余 {count} 天"`
- 支持复数：`"有 {count, plural, =0 {无文件} =1 {1个文件} other {#个文件}}"`
- 支持选择：`"{gender, select, male {他} female {她} other {他们}}"`

### 命名空间

按模块划分命名空间，避免命名冲突：

```json
{
  "common": {},
  "auth": {},
  "user": {},
  "order": {},
  "product": {},
  "payment": {},
  "error": {}
}
```

## 动态加载

### 按路由/模块加载

- 按需加载语言包，减少初始包体积
- 路由切换时按需加载对应语言包
- 首页预加载默认语言，语言包懒加载

### 语言检测顺序

1. URL 参数：`/en-US/dashboard`
2. 用户偏好：用户上次选择的语言
3. Cookie：`locale=zh-CN`
4. 浏览器语言：`Accept-Language`
5. 默认语言：兜底到 `zh-CN`

## 日期/时间/数字本地化

### 日期格式

| 地区 | 格式 | 示例 |
| --- | --- | --- |
| zh-CN | YYYY-MM-DD | 2024-01-15 |
| en-US | MM/DD/YYYY | 01/15/2024 |
| ja-JP | YYYY/MM/DD | 2024/01/15 |

使用 `Intl.DateTimeFormat`：

```typescript
new Intl.DateTimeFormat('zh-CN', { year: 'numeric', month: '2-digit', day: '2-digit' }).format(new Date())
```

### 数字格式

```typescript
// 千分位
new Intl.NumberFormat('zh-CN').format(1234567)  // 1,234,567

// 货币
new Intl.NumberFormat('zh-CN', { style: 'currency', currency: 'CNY' }).format(100)  // ¥100.00
new Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' }).format(100)  // $100.00
```

### 百分比

```typescript
new Intl.NumberFormat('zh-CN', { style: 'percent' }).format(0.25)  // 25%
```

## RTL 适配

支持阿拉伯语、希伯来语等 RTL 语言时：

- CSS 使用 `dir="rtl"` 自动翻转布局
- 使用 `logical properties`（`margin-inline-start` 而非 `margin-left`）
- 图标方向需要手动适配（如箭头方向）
- 文字对齐方式自动适配

## 测试要求

- 所有页面切换语言后必须视觉验证
- 边界文本长度（长文本不截断、不溢出）
- 动态内容（用户输入、API 返回）不参与翻译
- 数字、日期格式在所有支持语言下正确显示

## 翻译工作流

```
中文原文 → 翻译工具/人工翻译 → 翻译审校 → 合入语言包
                                    ↓
                              译文 QA → 修复问题 → 重新合入
```

翻译完成后必须有翻译审校，禁止直接使用机器翻译作为最终文本。

## 禁止行为

- 禁止在代码中写死中文（除技术术语和日志）
- 禁止拼接翻译文本（"已选择" + count + "项" → 模板插值）
- 禁止假设文本长度（不同语言文本长度差异大，必须留足空间或用省略）
- 禁止不同语言使用不同图片（需要替换的图片必须有对应语言版本）
- 禁止用户可见的错误信息不翻译
- 禁止数字、日期、货币格式不本地化
