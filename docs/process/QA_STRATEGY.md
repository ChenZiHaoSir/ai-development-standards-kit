# QA Strategy

## 缺陷分级

| 等级 | 定义 | 响应 |
| --- | --- | --- |
| P0 | 数据丢失、密钥泄露、应用无法启动、严重安全事故 | 30 分钟内响应，24 小时内修复、回滚或阻断发布 |
| P1 | 核心流程不可用、保存失败、导出失败、关键任务无法完成 | 2 小时内响应，3 个工作日内处置 |
| P2 | 重要功能异常但有规避方式 | 1 个工作日内响应，当前或下个迭代排期 |
| P3 | 视觉、文案、轻微交互问题 | 3 个工作日内响应，按优先级进入优化池 |

## 验证证据

每次 QA 结论必须包含：

- 测试范围
- 测试环境
- 执行命令
- 关键日志或截图
- 通过/失败判断
- 未覆盖风险

## UI 验证

- 覆盖空、加载、成功、失败和离线状态。
- 覆盖主要断点或窗口尺寸。
- 文本、按钮、卡片、表格和弹窗不得重叠。
- 图标按钮必须有 tooltip 或可访问标签。
- 不依赖颜色单独表达状态。
