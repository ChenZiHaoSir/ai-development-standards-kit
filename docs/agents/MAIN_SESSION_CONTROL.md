# Main Session Control

主会话是项目唯一的沟通入口和控制台。用户只需要和主会话交流，所有子会话只执行主会话分配的任务。

## 主会话职责

- 理解用户目标、约束、优先级和验收标准。
- 将项目拆成可执行任务，并判断哪些任务可以并行。
- 为每个专业子智能体分配清晰目标、输入、输出和验收标准。
- 维护项目状态看板：阶段、任务、负责人、状态、阻塞、风险、下一步。
- 接收子智能体结果，整合冲突，去重建议，决定最终执行顺序。
- 对用户只汇报高信号内容：进度、风险、决策点、验证结果和下一步。
- 负责最终质量门禁，不能把子智能体的结论原样当作最终结论。

## 主会话禁止事项

- 禁止让用户同时和多个子会话沟通项目方向。
- 禁止把未拆清楚的模糊任务直接派给子智能体。
- 禁止让多个子智能体同时修改同一文件、同一模块或同一接口契约。
- 禁止声称子智能体已经执行，除非运行环境真的创建并执行了对应任务。
- 禁止跳过整合阶段，直接拼接多个子智能体输出。

## 主会话固定输出

每次关键阶段更新时，主会话应维护以下信息：

```markdown

- 项目目标：
- 当前阶段：
- 当前里程碑：
- 总体状态：正常 / 有风险 / 阻塞
```

## 决策规则

- 用户方向 > 项目规范 > 主会话判断 > 子智能体建议。
- 涉及范围扩大、技术栈改变、安全豁免、跳过测试、发布风险时，必须回到主会话由用户确认。
- 子智能体建议冲突时，主会话按用户目标、架构边界、安全风险、验证证据和维护成本裁决。
