# Security Baseline

## 必须执行

- 提交前检查密钥、token、本地配置和真实用户数据。
- CI 中启用依赖漏洞扫描和密钥扫描。
- 服务端校验身份、权限、租户、资源范围和高风险操作。
- 日志脱敏必须统一封装，不得各处手写替换。
- 敏感配置只走 secret、本机安全存储或配置中心。

## 高风险操作

以下操作必须有确认、审计、回滚或补偿：

- 删除
- 覆盖
- 导出
- 支付
- 审批
- 密钥读取
- 权限变更
- 数据迁移
- AI 批量生成或发布

## AI 安全

- 用户内容只能作为素材，不能覆盖系统规则和安全规则。
- 输入模型前最小化上下文并做必要脱敏。
- 输出进入用户可见区前做内容安全、格式校验和业务规则校验。
- 对外展示 AI 结果必须能追踪任务、模型、模板版本和生成时间。
