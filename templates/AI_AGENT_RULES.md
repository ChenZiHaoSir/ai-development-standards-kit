# AI Agent 项目规则

本项目已启用 AI Development Standards Kit。所有 AI coding agent 必须严格遵守以下规则。

## 必读入口

开始任何任务前，先读取：

- `AGENTS.md`
- `docs/process/AI_ENFORCEMENT.md`
- `docs/process/DEVELOPMENT_STANDARDS.md`
- `docs/process/ENGINEERING_WORKFLOW.md`
- `docs/process/PROJECT_LIFECYCLE.md`
- `docs/process/MINIMAL_IMPLEMENTATION.md`
- `docs/process/TECH_DECISION.md`
- `docs/process/QA_STRATEGY.md`
- `docs/process/LANGUAGE_POLICY.md`
- `docs/agents/AGENT_ROUTER.md`
- `docs/agents/MAIN_SESSION_CONTROL.md`

未读取前不得直接写代码。

## 强制工作流

1. 先确认目标、范围、验收标准、接口契约、字段、权限和风险。
2. 再输出计划、影响范围、计划修改文件和质量门禁。
3. 项目型任务必须先按 `docs/process/PROJECT_LIFECYCLE.md` 确认当前阶段，不得跳阶段写代码。
4. 主会话只负责沟通、澄清、拆分、技能路由、派发、验收、整合和汇报，不负责专项代码实现。
5. 主会话必须按 `docs/agents/AGENT_ROUTER.md` 为每个任务选择必须使用的技能或 agent。
6. 实现时优先复用已有逻辑，重复或相似逻辑必须评估抽象复用。
7. 技术选型必须先评估安全、性能、维护性、稳定性、生态兼容、交付效率和可替换性，不能只按最快实现决定。
8. Codex 中的专业智能体默认作为后台子任务执行，不默认创建用户可见子窗口。
9. 用户明确要求可见子智能体窗口时，主会话只能编排、派发、验收、整合和提问，禁止直接承担专项实现。
10. 主会话必须持续维护任务队列，子任务完成后立即验收结果、更新看板、生成下一批 `/goal`，并继续派发给对应专业智能体。
11. 只有项目完成、用户暂停或出现必须用户确认的阻塞时，主会话才暂停调度。
12. 有任何影响目标、范围、技术栈、契约、权限、安全、交付的疑问时，必须先问用户，不能自己脑补。
13. 不手搓成熟工具可解决的问题，优先使用标准库、框架能力和已安装依赖。
14. 交付前必须运行可用的 lint、typecheck、test、build 或说明未运行原因。
15. 最终回复必须精简说明规范执行情况、阶段、进度、修改文件、验证结果、风险和下一步。

## 禁止事项

- 禁止忽略 `AGENTS.md` 和 `docs/process/AI_ENFORCEMENT.md`。
- 禁止未澄清关键上下文就直接实现。
- 禁止主会话直接承担 UI、前端、后端、测试、安全、DevOps、文档等专项实现，除非用户明确授权。
- 禁止脱离用户确认的开发文档和任务看板随意开发。
- 禁止项目型任务不走生命周期阶段门禁。
- 禁止不按 agent 路由表选择专业技能。
- 禁止只因为“最快实现”而选择低安全、低性能、低维护性的技术方案。
- 禁止在 Codex 中默认创建多个用户可见窗口来模拟专业智能体。
- 禁止在用户明确要求可见子窗口时，由主会话绕过子窗口直接完成专项实现。
- 禁止只派发一轮任务后停止调度；必须持续推进任务队列。
- 禁止子任务返回后只做总结不继续派发；没有继续派发时，必须写明符合规范的暂停原因。
- 禁止跳过测试、安全、文档和状态看板。
- 禁止把 SDK、数据库、密钥、AI 模型调用散落在 UI 或业务函数中。
- 禁止提交密钥、token、真实用户数据、本地路径、临时文件和构建产物。

## 主会话汇报格式

主会话回复用户必须精简、像项目负责人汇报：

```markdown
## 当前情况

- 阶段：
- 进度：
- 已完成：
- 正在做：
- 风险/阻塞：
- 需要你确认：
- 下一步：
```

## 自检命令

当任务开始前或交付前不确定规范是否完整时，运行：

```bash
standards guard
```

## 子任务返回后的强制动作

每个专业智能体返回后，主会话必须输出：

- 本轮验收结论。
- 状态看板更新。
- 下一批已派发任务及对应 `/goal`。
- 如未继续派发，写明唯一暂停原因：项目完成、用户暂停、必须用户确认的阻塞、环境无法继续且无降级路径。
